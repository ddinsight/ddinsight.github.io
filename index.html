Introduction

What is DD insight?

Have you ever wondered how network performance is affecting users of your app? DD insight can pull you out of darkness and provides you with insight into network quality and possible problems your users are experiencing, letting you monitor and analyze network side of service quality so that you can proactively seek out solutions to problems and ways to service enhancement.

App providers can identify network problems with analysis data provided by DD insight and use the data to work with network providers to solve problems and enhance network quality for your users.

If you’re the provider of both the app and the network, then you don’t have to entirely rely on monitoring devices on you network which only provides network-side perspective. Rather, you will be able to acquire data on network quality users actually experiences since data is provided by mobile devices themselves. Using this data, you will be able to find ways to enhance both network and services your app provides.

How It Works

The following three major sub-projects comprise DD insight.

  1. dd-collector
    This is a module that collects and filters logs from smart devices and processes them using distributed system.

  2. dd-streamworks
    This consists of real-time processing of logs and scalable job server.

  3. dd-analytics
    This consists of the batch process module for saved logs and the presentation module.
    The presentation module is mainly for map-based visualization of output of QoE analysis based on results of batch process (Comprehensive presentation of results of real-time analysis can be provided in a commercial version)

Use Cases

As a Solution

Network monitoring

Based on logs obtained from mobile devices of app users all over the world, you can monitor what’s happening network-wise anywhere in the world.

Network actions

If there is a network problem to be addressed in a specific location or network segment, you can instantiate proper modules in your app to prevent or ease further problems, or if you have proper authority to control network itself, you can modify network configuration to address the network issue.

Customer experience management

You can not only provide proper measures for customers who’s already encountered the network issues but also provide proactive measures to prevent potential problems for customers deemed at risk of experiencing them.

Applicable Services

Multimedia streaming

Network shortage or service disruption during video/audio streaming playback can be monitored and analyzed by collecting logs during service intervals.

Game on mobile network

Network problems during playing of games using network connection can be monitored and analyzed by collecting corresponding logs.

Web-based application

Network problems during dynamic web contents loading can be monitored and analyzed.

Connected-car services

Network problems while using telematics services and infotainment services can be monitored and analyzed through collection of logs from corresponding devices.
